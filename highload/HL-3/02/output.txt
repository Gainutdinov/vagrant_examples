Задание:
1. Для выполнения задания необходимо создать такую инфраструтуру:
● ВМ1 mariadb со следующими параметрами: 1 CPU / 2G memory;
● ВМ2 wordpress со следующими параметрами: 1 CPU / 1G memory.
● ВМ3 jmeter со следующими параметрами: 1 CPU / 2G memory.
Примечание: возможно, потребуется обновить систему на ВМ[1-3].
2. Используя docker-compose.yml из задания «HL 12. Стенд для тестирования. Поиск метрик (Стенд на докере)», создайте:
○ docker-compose.yml на ВМ1 для запуска MariaDB с привязкой к внешнему IP;

```
docker-compose-mariadb.yml
```

○ docker-compose.yml на ВМ2 для запуска Wordpress.

```
docker-compose-wordpress.yml
mysql -h 54.69.229.238 -u wordpress -pwordpress
```

3. Запустите MariaDB и Wordpress в контейнерах.
4. Откройте стартовую страницу Wordpress в браузере, задайте параметры сайта, авторизуйтесь администратором.
5. Установите плагин для basic authentication в Wordpress и активируйте его.
6. Используйте для тестирования ВМ3 jmeter из задания «HL-3 01. Двухзвенная архитектура (приложение + БД). Нагрузочное тестирование». Измените значения IP-адреса и пути для тестирования. Установите значения Number of threads и Loop count равными 10. Запускайте тест, на каждом проходе увеличивая Number of threads и Loop count. Параллельно остлеживайте нагрузку на процессоры и память для ВМ1 mariadb и ВМ2 wordpress. Для каждого запуска теста фиксируйте значение Average.
7. Определите, при каком значении параметров Number of threads и Loop count время отклика системы превышает 3с.
8. Сравните полученные результаты с результатами из пердыдущего задания. Свои Выводы с результатами пришлите в ответе.
